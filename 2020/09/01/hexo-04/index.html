Hexo-nexT 4 部署

发表于 2020-09-01 分类于 Hexo

借助 Github Pages

远程仓库起名

  • 方式一
    • 形如 {username}.github.io
    • username 是你的用户名，忽略大小写
  • 方式二
    • 像一般仓库起名即可
    • 如 amusing

部署步骤

  1. 安装 git 部署插件

    • D:\hexoblog>cnpm install hexo-deployer-git --save

  2. 配置博客根目录的 _config.yml，这一段在最下面，拉到底即可

  3. 若仓库名不是 {username}.github.io，再回到顶部编辑 url 与 root

  4. 部署到 Github 的仓库里

    • D:\hexoblog>hexo d

  5. 如有需要，可以在本地仓库添加一条分支，将 Markdown 文件加到该分支，然后推送至远程仓库

  6. 在 GitHub 的相应仓库下

    • 选择 Settings
    • 下拉，找到 GitHub Pages
    • Branch -> master -> /(root) -> Save

博客网址

  • https://yorkfish.github.io/amusing

添加文章

方式一

  • 使用命令 $ hexo new [layout] <title>
    • new 可以用缩写 n
    • layout 默认是 post，可以在 _config.yml 中的 default_layout 更改
    • title 对应具体文章中 YAML 题头内的 title
  • 举例
    • D:\hexoblog>hexo n "article"
  • 新建文档的默认内容可以在博客根目录下的 scaffolds 文件夹的 post.md 中自定义

方式二

  • 使用命令生成的文章会添加到博客根目录的 source/_posts 文件夹中
  • 直接在 _posts 文件夹中新建文件也行，就是得自己写 YAML 题头

预览

  • 博客首页处，默认显示全文
  • 在文章中适当处加一行 <!--more-->，博客首页会有预览效果

写完后

  • 使用 hexo s 在本地预览

  • 若出现问题，建议使用 hexo s --debug

  • 检查无误后，三步走
